## 1. 项目概述

### 1.1 项目背景

上门按摩系统是一个基于O2O模式的本地生活服务平台，连接用户与专业按摩技师，提供便捷的上门按摩服务。系统支持用户在线预约、技师接单、实时追踪、在线支付、服务评价等完整业务闭环。

### 1.2 项目目标

- 构建高效、稳定、可扩展的上门服务平台
- 提供流畅的用户体验和技师工作流程
- 支持多城市、多门店、多角色的运营模式
- 实现智能化调度和精准推荐

### 1.3 系统角色

| 角色 | 说明 |
|------|------|
| 用户端APP | C端用户使用，预约服务、在线支付、评价反馈 |
| 技师端APP | 技师使用，接单、服务流程管理、收入管理 |
| 商家端后台 | 门店/代理商使用，技师管理、订单管理、数据统计 |
| 平台管理后台 | 平台运营使用，全平台管理、财务结算、风控管理 |

## 5. 数据库设计

### 5.1 数据库选型

**主数据库：PostgreSQL 16+**

选择理由：
- 支持JSON类型，灵活存储非结构化数据
- 支持PostGIS扩展，强大的地理信息处理能力
- 优秀的并发性能和事务支持
- 开源免费，社区活跃

**缓存数据库：Redis 7.2+**

选择理由：
- 高性能缓存
- 支持地理位置操作（GEO）
- 支持消息队列、分布式锁
- 支持持久化

### 5.2 核心数据表设计

#### 5.2.1 用户表 (users)

```sql
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    phone VARCHAR(20) NOT NULL UNIQUE,
    password_hash VARCHAR(255),
    nickname VARCHAR(50),
    avatar VARCHAR(500),
    gender SMALLINT DEFAULT 0, -- 0:未知 1:男 2:女
    birthday DATE,
    email VARCHAR(100),
    
    -- 会员信息
    member_level SMALLINT DEFAULT 0, -- 会员等级
    member_points INTEGER DEFAULT 0, -- 积分
    member_expire_at TIMESTAMP,
    
    -- 状态
    status SMALLINT DEFAULT 1, -- 1:正常 0:禁用
    last_login_at TIMESTAMP,
    last_login_ip VARCHAR(50),
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP
);

CREATE INDEX idx_users_phone ON users(phone);
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_created_at ON users(created_at);
```

#### 5.2.2 技师表 (technicians)

```sql
CREATE TABLE technicians (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    
    -- 基本信息
    real_name VARCHAR(50) NOT NULL,
    id_card VARCHAR(20) NOT NULL,
    gender SMALLINT NOT NULL,
    birthday DATE,
```

## 8. 安全设计

### 8.1 认证与授权

- JWT Token认证
- Token过期时间：2小时
- Refresh Token过期时间：7天
- 支持微信、支付宝第三方登录
- 短信验证码登录

#### 8.1.2 权限控制

- RBAC权限模型
- 接口级权限控制
- 数据级权限控制（城市、门店）

### 8.2 数据安全

#### 8.2.1 敏感数据加密

- 密码：bcrypt加密
- 身份证号：AES加密存储
- 手机号：脱敏显示
- 银行卡号：AES加密存储

#### 8.2.2 传输安全

- 全站HTTPS
- 接口签名验证
- 防重放攻击（时间戳+nonce）

### 8.3 业务安全

#### 8.3.1 防刷机制

- 短信验证码：1分钟1条，每天最多10条
- 接口限流：基于IP和用户ID
- 订单限流：同一用户同一时间段只能有1个进行中的订单

#### 8.3.2 风控规则

- 异常订单检测：短时间内多次取消
- 虚假定位检测：技师定位异常跳变
- 刷单检测：同一用户频繁下单同一技师

### 8.4 隐私保护

- 用户隐私协议
- 服务协议
- 数据脱敏
- 日志脱敏

| Java | 生态成熟、企业级支持好 | 内存占用高、启动慢 | 备选 |
| Node.js | 开发效率高、前后端统一 | CPU密集型性能差 | 不推荐 |
